# ai-quant-assembler

## Overview

ai-quant-assembler is an AI‑augmented, plug‑in‑driven quantitative pipeline targeting Chinese A‑share markets. The system accepts a REST request, schedules a job, pulls market data through extensible plugins, runs a strategy generated by a large language model, backtests the strategy, and returns equity‑curve artefacts. All heavy lifting runs in background threads so the FastAPI endpoint remains fully asynchronous. Persistence is provided by SQLite for job metadata and Parquet for result caching, enabling zero‑ops Docker deployment.

## Features

- **Plugin architecture** – third‑party data sources register via setuptools entry‑points.
- **LLM‑driven strategy generation** – safe DSL expressions are produced from natural‑language prompts.
- **Background execution** – jobs run in separate threads, keeping the API responsive.
- **Result caching** – back‑test results are stored as Parquet files for fast retrieval.


## Quick Start

```bash
# Start the API server
uvicorn src.api.router:api_router --host 0.0.0.0 --port 8000
```

Submit a job via HTTP:

```bash
curl -X POST http://localhost:8000/jobs \
  -H "Content-Type: application/json" \
  -d '{"symbol": "600519.XSHG", "start": "2023-01-01", "end": "2023-12-31", "prompt": "Long only strategy based on moving average crossover"}'
```

## Architecture

```
+-------------------+      +-------------------+      +-------------------+
|   FastAPI Router  | ---> |  JobScheduler     | ---> |  PipelineEngine   |
+-------------------+      +-------------------+      +-------------------+
                                 |                         |
                                 v                         v
                         SQLite (jobs.db)          ResultCache (Parquet)
```

The **FastAPI router** receives HTTP requests and forwards them to the **JobScheduler**, which persists job metadata in SQLite. The scheduler launches a background thread that runs the **PipelineEngine**. The engine generates a strategy via the **LLMClient**, fetches market data through registered **DataProviderPlugin** implementations, back‑tests the strategy, and stores the equity curve in the **ResultCache**.

## API Reference

### POST `/jobs`
Creates a new back‑test job.

**Request body** (`application/json`):
```json
{
  "symbol": "string",
  "start": "ISO‑8601 datetime",
  "end": "ISO‑8601 datetime",
  "prompt": "string"
}
```
**Responses**:
- `202 Accepted` – returns the created job object.
- `400 Bad Request` – validation errors.

### GET `/jobs/{job_id}`
Retrieves job metadata.

**Responses**:
- `200 OK` – job JSON.
- `404 Not Found` – unknown job identifier.

### GET `/jobs/{job_id}/result`
Returns the back‑test result (equity curve) for a completed job.

**Responses**:
- `200 OK` – JSON containing the equity curve.
- `404 Not Found` – result not yet available.
